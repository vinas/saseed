Installation

SaSeed is super easy and fast to install. You don't need to be an expert or anything like it. =)

There is however one thing you should understand about it's basic structure: SaSeed is designed to keep both the framework's and your application's files out of the server's public folder.

The goal of doing this is to prevent your files to be directly accessed (and thus downloaded). It's a security measure to protect sensitive information those files should contain, such as database information and passwords.

That said, it's time to see how it is structured.

SaSeed's basic package contains 3 main folders:

  • Application
  • public_html
  • SaSeed
Both Application and SaSeed are at the same level as the public folder, which is the context root folder, the one accessed through a URL request.

Example: when accessing http://saseed.pe.hu/, the user will be accessing public_html/index.html.

For this tutorial, we are supposing you are working in a Windows environment and have already downloaded XAMPP. If you haven't go ahead and do it now. XAMPP is a great easy-to-use apache server that will do everything you need.

If you are running Linux or using a different server, chances are that have a bit more experience and will need very little explanation, so keep following these instructions and you'll understand what you have to do to have it set. ;)

Once installed, the default folder for XAMPP is C:\xampp\. It's context root folder will then be C:\xampp\htdocs\.

Inside htdocs we are going to create our application folder. Let's name it 'myApp'.

  • C:\xampp\myApp\

Once that is done, we should copy the 3 main folders to C:\xampp\myApp\. After copying them, we shall have the main folders at the following addresses:

  • C:\xampp\myApp\Application
  • C:\xampp\myApp\public_html
  • C:\xampp\myApp\SaSeed
And... That's basically it. You have installed it!! =D

But don't get to excited yet! Although it is already installed, it is still not working, for it cannot be accessed through an URL request. If you start you server, you'll notice that when you access http://localhost/, the folder C:\xampp\htdocs is being considered the document root.

To change that we will need to change XAMPP's document root.

THEN it will be ready! =)
